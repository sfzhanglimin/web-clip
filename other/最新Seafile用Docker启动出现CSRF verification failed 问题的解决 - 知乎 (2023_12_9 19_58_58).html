最新Seafile用Docker启动出现CSRF verification failed 问题的解决

跋涉者
创造梦想的软件工程师
1 人赞同了该文章

根据官方文章：
用 Docker 部署 Seafile 服务 - seafile-manual-cn
重新安装了Seafile,安装完成后，登录页面可打开，输入密码后报以下错误：
seafile CSRF verification failed. 用F12打开查看是403错误。

在网上找了一些文章，有些说是https的问题，需要去修改nginx的配置，但是我确定我没有运行https,且测试修改nginx也不行，最后确定以下方法是有效的：

以下命令：

#进入seafile 容器
docker exec -it seafile /bin/bash

#进入conf配置目录

cd conf

#编辑这个python文件

vi seahub_settings.py

#在最后添加一行：

CSRF_TRUSTED_ORIGINS = ["http://你的域名:端口"]
#这里要注意，网上很多教程都没说要输入前面那个http://，经我实测，如果不加这个http://，问题还是得不到解决。同时要注意后面还需要端口

然后按ESC，wq 保存这个文件,再退出容器

exit

#重启docker

docker stop seafile

docker start seafile

这样再检查一下应该就可以了
